What this prototype proves — and what's deferred. Two blends run at once (your Regular and Premium cases) and each is a tracked record, not a screenshot. Every value is retrieved, not typed: a case is promoted from a planner row, and the operator only swaps estimates for actuals (TOV after the receipt, the three sample RVPs). The Validate stage is the scenario tool — when the truck fraction clears 0.8 it routes to PQ with every option priced against resultant RVP and uncaptured margin, exactly as you'd work it by hand. Margin is recorded both ways (round-up captured, late-truck refused). The folder builds itself stage by stage into SharePoint › Quality › Blends › {Blend ID}. The two-tank rule is enforced: the board blocks a second Premium blend while one is open. Now wired: cases load from and persist to the terminal server (append-only), with device check-out/check-in for control-room↔iPad handoff and QC force-release of a stranded checkout. Still deferred to IT: the automated order email on "Book trucks" and the SharePoint folder writes. Auto-order hard rules are placeholders pending your team's review — today only the 0.8-fraction → PQ rule is wired.
